LED Timecode Panel

Build report — the design, and the hardware working.

Yubo Huang · updated 2026-06-27 22:29 MDT

← Full design & build log

A row of LEDs shows a fast-advancing timecode — a number that ticks up many times a second. Every camera pointed at the panel can read that number off each frame it captures, so comparing what two cameras read at the same moment shows how far apart their shutters actually fired — their synchronization error.

Blink one LED

The smallest possible test: the microcontroller (a Raspberry Pi Pico) drives one LED through one resistor to ground. If it blinks, the code, the board, and the wiring are all good — a foundation to build on.

The circuit

To drive several LEDs from just a few pins, the Pico feeds a shift-register chip — it turns three control wires into eight independent on/off outputs. Seven of those outputs each light an LED through a 240 Ω resistor, and the seven on/off states together form one number: the timecode.

That number is shown in Gray code — a counting scheme in which only one LED changes from one step to the next. So a camera frame that happens to catch a change still reads a clean value (off by at most one), never a scrambled one.

Glossary

Every term and label that appears above, in plain language.

TermMeaning
MicrocontrollerA tiny computer on a single chip. Here it’s a Raspberry Pi Pico, which runs the code that drives the LEDs.
Raspberry Pi PicoThe microcontroller board used — the “controller” in the diagrams.
BreadboardA reusable board for wiring circuits by pushing parts into rows of holes — no soldering.
LEDLight-emitting diode — the small lights that show the code.
Resistor (240 Ω)Limits the current through an LED so it lights safely; Ω (ohm) is the unit of resistance.
Shift register (74HC595)A chip that turns a few control wires into many on/off outputs — here, three wires from the Pico into eight outputs. The “driver chip.”
TimecodeA number that counts up at a steady, known rate. Read it off a camera frame and you know when that frame was captured.
Gray codeA way of counting in which only one bit (one LED) changes between consecutive values — so a frame catching a mid-change still reads a clean value, off by at most one.
GP15, GP17–GP19Numbered general-purpose pins on the Pico. GP15 drives the single blink LED; GP17–GP19 carry the three control signals to the shift register.
SER, SRCLK, RCLKThe shift register’s three control inputs — SER: data in; SRCLK: shift clock (loads each bit); RCLK: latch clock (shows the whole byte on the outputs at once).
QA, QB–QHThe shift register’s eight outputs. QB–QH each light one LED (seven in all); QA is unused.
OE, MROutput Enable and Master Reset — tied to the levels that keep the outputs on and stop the chip from resetting.
VCC, GND, 3V3Power and ground: VCC = the chip’s supply; GND = 0 V (the common return); 3V3 = the Pico’s 3.3 V output, which powers the chip.
